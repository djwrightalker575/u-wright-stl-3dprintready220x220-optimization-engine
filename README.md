# Creality 220 Orientation Optimizer (Local Operator UI)

Brand: **U-Wright Open Innovations**

## Features
- FastAPI + SQLite local-only operator interface
- Background run execution with automatic mock fallback when real slicer CLI is unavailable
- Pause/resume/stop controls with persistent run/model history
- Candidate-based slicing semantics with strict SUP0/SUP1 pairing and deterministic candidate ids
- Data quality gates that invalidate physically inconsistent support deltas
- Per-candidate visual previews (`iso`, `top`, `supports_overlay`) shown in run detail

## Install
```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

## Start UI
```bash
orienter ui
```
Server: http://localhost:8787

## Why negative support deltas happened (and fix)
Historically, negative support deltas can occur when SUP0/SUP1 files are mismatched, overwritten, or cache-collided.
This build addresses that by:
- hard-forcing support toggles for each mode (`support_material`, `support_material_auto`, and threshold) via explicit slicer args
- using deterministic `candidate_id = sha256(model_hash + orientation_id + profile_id)`
- writing unique artifacts per support mode:
  - `...__cand_<n>__<candidate_id>__SUP0.gcode`
  - `...__cand_<n>__<candidate_id>__SUP1.gcode`
- using cache keys that include support mode: `(model_hash, orientation_id, profile_id, support_mode)`
- strictly parsing slicer stats from headers:
  - `filament used [mm]` (sum all extruders)
  - `filament used [g]` (sum all extruders)
  - `estimated printing time (normal mode)`
- applying a hard invalidation gate:
  - invalidate candidate if `support_filament_delta < -0.5 mm` or `support_time_delta < -30 s`

## Preview generation
Per candidate previews are generated at:
```
<run>/previews/<model>/<candidate_id>/
  iso.png
  top.png
  supports_overlay.png
```
Notes:
- `iso.png` and `top.png` are lightweight local previews.
- `supports_overlay.png` is generated by parsing support toolpath sections from gcode comments (`;TYPE:Support...`) and projecting XY.
- In environments without full 3D rendering stacks, previews are intentionally dependency-light and deterministic.
